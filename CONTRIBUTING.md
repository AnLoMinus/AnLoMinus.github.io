# מדריך תרומות

תודה על הרצון לשפר את AnLoMinus Studio! להלן השלבים המומלצים:

## 1. פתיחת Issue
- תארו את הבעיה/הצעה בעברית או באנגלית.
- צרפו צילומי מסך/קישורים במידת האפשר.
- הציעו פתרון קצר כדי להבין כיוון כללי.

## 2. Fork + Branch
```bash
git checkout -b feature/short-title
```
- שמרו ענפים קטנים וממוקדים.
- ודאו ש־`main` מעודכן לפני פתיחת PR.

## 3. סגנון קוד
- HTML: שמות מחלקות BEM ידידותיות.
- CSS: שימוש ב־custom properties קיימות.
- JS: קוד ES2018+, פונקציות קצרות, ללא תלותים חיצוניים כבדים.

## 4. בדיקות לפני PR
- הריצו Lighthouse או בדיקות ביצועים מקומיות.
- בדקו התאמות מובייל (375px) ודסקטופ (1440px).
- הריצו `npm run lint` (כשתתווסף תצורה).

## 5. Pull Request
- תארו מה נעשה ומדוע.
- קשרו ל־Issue רלוונטי.
- ציינו השפעות UX/NLP/n11n אם קיימות.

## 6. קהילתיות
- שמרו על תקשורת פתוחה ומכבדת (ראו `CODE_OF_CONDUCT.md`).
- אם אתם לא בטוחים, שאלו לפני שקופצים לפתרון מורכב.
